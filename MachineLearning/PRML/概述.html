1 机器学习的定义¶

机器学习算法是一种能够从数据中学习的算法。实际上，即使是在机器学习的专业人士中，也不存在一个被广泛认可的定义来准确定义机器学习是什么或不是什么，但专业人士有尝试着去下一些定义，下面我们给出两个示例：

  • Machine Learning is the field of study that gives computers the ability to learn without being explicitly programmed. — Arthur Samuel, 1959
  • A computer program is said to learn from experience $E$ with respect to some task $T$ and some performance measure $P$, if its performance on $T$, as measured by $P$, improves with experience $E$. — Tom Mitchell, 1997

下面，我们希望对经验$E$、任务$T$、性能度量$P$有一个直接的解释。

  • 任务$T$
    机器学习可以让我们解决一些认为设计和使用确定性程序很难解决的问题。从科学和哲学的角度来看，机器学习之所以受到关注，是因为提高我们对机器学习的认识需要提高我们自身对智能背后原理的理解。
    从“任务”的相对正式的定义上说，学习过程本身不能算是任务。学习是我们所谓的获取完成任务的能力。例如，我们的目标是使机器人能够行走，那么行走便是任务。我们可以变成让机器人学会如何行走，或者可以人工编写特定的指令来指导机器人如何行走。 通常机器学习任务定义为机器学习系统应该如何处理样本。样本是指我们从某些希望机器学习系统处理的对象或事件中收集到的已经量化的特征的集合。我们通常会将样本表示成一个向量$\boldsymbol{x} \in \mathbf{R}^n$，其中向量的每一个元素$x_i$是一个特征。例如，一张照片的特征通常是指这张图片的像素值。
  • 性能度量$P$
    为了评估机器学习算法的能力，我们必须设计其性能的定量度量。通常性能度量$P$是特定于系统执行的任务$T$而言的。
    对于诸如分类、缺失输入分类和转录任务，我们通常度量模型的准确率。准确率是指该模型输出正确结果的样本比率。我们也可以通过错误率得到相同的信息，错误率是指该模型输出结果错误的样本比率。
    通常，我们会更加关注机器学习算法在未观测数据上的性能如何，因为这将决定其在实际应用中的性能。因此，我们用测试集数据来评估系统性能，将其与训练机器学习系统的训练集数据分开。
  • 经验$E$
    经验是指在学习过程中发现的某些结构、性质、规律等，根据学习过程中的不同经验，机器学习算法可以大致分类为无监督算法和监督算法。

2 机器学习的作用¶

我们都知道，传统数据分析也能挖掘出数据中的一些规律。例如垃圾邮件分类问题，广告邮件中一般会出现“推荐”、“免费”等词语，我可以利用传统数据分析，对带有这些词语的邮件进行筛选，那么就完成了垃圾邮件的分类。但是，广告营销者也会观察到这一问题，他们会更改邮件中的这些词语，转而使用另一些词语代替，例如把“免费”改为“福利”等。对于这一情况，传统数据分析可以通过在算法中加入这些词来达到修改算法的目的，但是，随着时间的推移，这些需要加入的词语就会越来越多，人力很难实现。
这时候，机器学习就起到非常重要的作用了。机器学习算法会自己去学习这些词语，对算法进行优化，就算广告营销者把邮件中的“免费”改为了“福利”，机器学习算法也会学得这一改变，从而做出垃圾邮件的判断，这就大大提高了工作效率。

3 机器学习的类别¶

3.1 Supervised, Unsupervised, Semisupervised, and Reinforcement Learning¶

  • Supervised Learning
    监督式学习，即对于训练集中的每一个数据，都有对应的标签。
  • Unsupervised Learning
    无监督式学习，对于训练集中的数据，没有对应的标签，我们需要单纯从数据中发现规律。
  • Semisupervised Learning
    半监督式学习，顾名思义，数据集中的一部分数据没有对应的标签。
  • Reinforcement Learning
    强化学习，善于控制能够在某个环境下自主行动的个体，通过和环境的互动不断改进它的行为。

3.2 Online and Batch Learning¶

  • Online Learning
    在线学习，即在对输入进行处理的时候也在不断学习。可以一次只输入一个样本，利用输入的一个样本，计算输入结果，并和目标结果做残差，然后利用残差修正网络参数，紧接着就对下一个样本进行训练，然后继续修正权值，就这样依次做下去。由于每次只用一个样本进行训练，使得输入很容易受噪声影响，使得每次迭代不一定朝最优的方向，但这样的情况有时可以避免网络陷入局部最优。
  • Batch Learing
    批量学习，这是一种离线的学习方法，每次网络的训练都是用所有的输入样本。针对一个训练样本，我们可以比较网络输出和目标输出之间的残差，在求出所有样本对应的残差之后，对残差去平均，得到整个训练样本的残差，然后利用这个残差求取梯度，进而对网络参数进行修正。这种方法能找出网络的最优解，而且由于是对所有样本取平均，所以可以消除噪音对网络的影响。但在网络规模很大，训练样本很多的情况下，这种方法计算非常耗时。

3.3 Instance-based and Model-based learning¶

  • Instance-based Learning
    基于实例的学习，学习系统通过实例进行学习，然后推广到新的用例中。
  • Model-based Learning
    基于模型的学习，从一系列数据集中抽象出一个模型，然后利用这个模型对新的用例做出预测。

4 机器学习的挑战¶

简而言之，我们的主要任务是选择学习算法并将其在数据集上训练这一算法，显然，可能出错的两个步骤是算法不好或者数据不好。从数据层面来看，我们会遇到训练数据量不足、训练数据不具代表性、数据质量不高等困难；而从算法层面上看，会遇到抽取的特征不好、过拟合、欠拟合等困难。后续我们会讲述如何在机器学习过程中克服这些困难。
没有免费午餐定理表明，在所有可能的数据生成分布上平均之后，每一个分类算法在未事先观测的点上都有相同的错误率。换言之，在某种意义上，没有一个机器学习算法总是比其他的要好。我们能够设想的最先进的算法和简单地将所有点归为同一类的简单算法有着相同的平均性能。幸运的是，这些结论仅在我们考虑所有可能的数据生成分布时才成立。在真实实际应用中，如果我们对遇到的概率分布进行假设，那么可以设计在这些分布上效果良好的学习算法。这意味着机器学习研究的目标不是找一个通用的学习算法或是绝对最好的学习算法，而是理解什么样的分布与人工智能获取经验的“真实世界”相关，以及什么样的算法在我们关注的数据生成分布上效果最好。

5 深度学习¶

在人工智能的早期，那些对人类智力来说非常困难、但对计算机来说相对简单的问题得到迅速解决，比如，那些可以通过一系列形式化的数学规则来描述的问题。人工智能的真正挑战在于解决那些对人来说很容易执行、但很难形式化描述的任务，如识别人们所说的话或图像中的脸。对于这些问题，我们人类往往可以凭借直觉轻易地解决。
针对这些比较直观的问题，我们可以讨论一种解决方案。该方案可以让计算机从经验中学习，并根据层次化的概念体系来理解世界，每个概念则通过与某些相对简单的概念之间的关系来定义。让计算机从经验获取知识，可避免由人类来给计算机形式化地指定它需要的所有知识。层次化的概念让计算机构建较简单的概念来学习复杂概念。如果绘制出表示这些概念如何建立在彼此之上的图，我们将得到一张“深”（层次很多）的图。基于这个原因，我们称这种方法为AI深度学习。
